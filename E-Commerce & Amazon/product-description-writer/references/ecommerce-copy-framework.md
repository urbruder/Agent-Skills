# Ecommerce Copy Framework

Use this reference for Amazon listings, ecommerce product pages, catalog batches, and product family consistency.

## Feature-to-benefit translation

Convert raw features into buyer outcomes:

- Material -> durability, comfort, look, care, or use case
- Size -> fit, storage, portability, coverage, capacity
- Compatibility -> confidence before purchase
- Design -> ease of use, style, speed, convenience
- Included items -> completeness, value, readiness
- Care instructions -> maintenance confidence

Pattern:

**Feature:** [what it is]  
**Benefit:** [why the buyer cares]  
**Copy:** [benefit-led sentence]

## Amazon title guidance

A strong Amazon title usually includes:

1. brand if provided
2. primary keyword
3. defining feature
4. use case or audience
5. size, color, quantity, compatibility, or variant if relevant

Do not overstuff. Prioritize readability.

## Amazon bullet strategy

Use five bullets with different jobs:

1. primary outcome or use case
2. differentiating feature
3. material, quality, or durability
4. compatibility, fit, or usability
5. gift, care, package contents, or purchase confidence

## Ecommerce PDP structure

A strong PDP should include:

1. clear title
2. short benefit description
3. scannable bullets
4. detailed description
5. specs
6. care or usage notes if relevant
7. FAQ for objections
8. SEO metadata

## Catalog batch consistency

For multiple SKUs:

- keep title structure consistent
- use the same bullet order across related products
- vary only product-specific attributes
- keep claims consistent across the product family
- avoid making one SKU sound superior unless it is actually positioned that way

## Product family title pattern

[Brand] [Product Type] - [Key Feature], [Material/Size/Compatibility], [Use Case], [Variant]

Example structure only. Do not invent details.

## Buyer objections to answer

Common ecommerce objections:

- Will it fit?
- Is it compatible?
- Is it durable?
- Is it easy to use?
- What is included?
- How do I care for it?
- Is it worth the price?
- Is it safe for my use case?
- How is this different from alternatives?

## SEO metadata

### Meta title
Keep concise. Include product type and primary keyword naturally.

### Meta description
Summarize benefit, differentiator, and use case. Avoid overpromising.

## Claim safety

Use cautious wording when proof is missing:

- designed for
- made to help
- ideal for
- built for
- helps keep
- supports
- crafted with

Avoid absolute claims unless verified:

- guaranteed
- best
- safest
- cures
- prevents
- certified
- non-toxic
- organic
- medical grade

## Review mining

When reviews are provided, extract:

- repeated buyer language
- praised benefits
- common objections
- unexpected use cases
- defects or concerns to avoid overclaiming

Do not quote reviews unless the user asks and provides permission or source text.
