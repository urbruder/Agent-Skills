# Spec Framework

## Spec sections

- summary
- problem
- users
- use cases
- MVP scope
- non-goals
- user flows
- screens
- data model
- integrations
- permissions
- edge cases
- acceptance criteria
- analytics
- testing
- rollout plan

## MVP rules

Cut anything that does not prove the core workflow. Prefer one user, one painful job, one clear output.

## Browser extension specifics

Include:
- manifest version
- permissions
- content scripts
- background/service worker
- popup UI
- storage
- host permissions
- privacy notes

## Internal tool specifics

Include:
- roles
- data sources
- CRUD actions
- approval states
- audit log
- export needs

## AI coding handoff

Make requirements testable. Avoid vague words like simple, easy, robust unless paired with measurable criteria.
