# Hook & Headline Machine

Generate 25 strong hooks or headlines for a business topic, asset, offer, or idea.

## Workflow

1. Identify the business context from the user's prompt:
   - audience
   - offer, topic, or asset
   - desired action
   - pain point or aspiration
   - level of audience awareness
   - platform or use case

2. If key context is missing, make reasonable business assumptions and state them briefly.

3. Generate exactly 25 hooks unless the user requests another number.

4. Make the set varied. Include problem-aware, outcome-driven, contrarian, curiosity, authority, data-style, mistake, myth, how-to, before/after, and executive insight hooks.

5. Select the best 5 to test first and explain why.

## Default output

```markdown
## Assumptions
- Audience: ...
- Use case/platform: ...
- Desired action: ...

## 25 Hooks & Headlines
1. ...
2. ...
...
25. ...

## Best 5 to Test First
1. ... — ...
2. ... — ...
3. ... — ...
4. ... — ...
5. ... — ...
```

## Quality rules

- Prioritize clarity, credibility, specificity, and business usefulness.
- Avoid empty hype, exaggerated urgency, and misleading clickbait.
- Make each hook usable immediately.
- Keep hooks concise unless the user asks for long-form headlines.
- Default to polished business language: clear, confident, direct, and useful.
